# /build-page <route>

Реализовать (или доработать) одну страницу до продакшен-качества по Figma-first.

## Шаги
1. Открыть `docs/08` для роута из `$ARGUMENTS` — перечитать секции страницы + общий layout.
2. **Figma MCP (docs/09):** снять точные размеры/цвета/шрифты/отступы/состояния для этого экрана. MCP недоступен → остановиться, сообщить пользователю, не выдумывать.
3. Определить нужные вызовы API (docs/03); убедиться, что есть типизированные функции в `shared/api` + thunks в нужном слайсе (docs/07); создать, если нет.
4. Собрать страницу на переиспользуемых `shared/ui` (shadcn) и `entities/*` — не дублировать карточки/кнопки/инпуты. Логика — в `hooks/`, не в JSX.
5. Применить токены docs/04 (цвета, Poppins/Inter, радиус 4px, контейнер). Соблюсти layout Figma точно (порядок секций, бейджи, hover).
6. Все тексты — через `t()` (docs/05). Реализовать состояния **loading / success / error** + 204/empty.
7. Адаптив по mobile-фрейму Figma (docs/08).
8. `npm run type-check` + `npm run lint`; починить. Сообщить, что построено, и любые отклонения/пробелы API.

## Definition of done
- Роут рендерится, тянет реальные данные, все интерактивы работают.
- Нет `any`, нет axios в компонентах, нет ошибок в консоли.
- Визуально совпадает с Figma на desktop и mobile.
