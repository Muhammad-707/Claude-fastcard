# /stop — зафиксировать и завершить сессию

Ты завершаешь рабочую сессию по **Fastcart**. Сохрани всё, чтобы следующая сессия не потеряла контекст. По порядку.

## 1. Разобрать, что было сделано
По этому диалогу и реальным изменениям файлов определить:
- Какие шаги `docs/10` **завершены**, **начаты**, **частично сделаны**.
- Новые **блокеры**, **решения**, **открытые вопросы**.
- Точный следующий шаг, если продолжить прямо сейчас.

## 2. Обновить docs/10
- Поставить `[x]` всем полностью готовым и проверенным шагам.
- `[~]` — начатым, но незаконченным.
- Новые блокеры — в раздел «Блокеры» (`[!]` / `[?]`).
- Значимые решения — в «Журнал решений» (по строке, напр. «Карусели на embla»).

## 3. Обновить статус
В `docs/10` (и при желании в CLAUDE.md) отразить текущий шаг и следующий.

## 4. Записать лог сессии
Определить сегодняшнюю дату и номер сессии (посчитать существующие `SESSION-*` за сегодня, +1).
Создать `.claude/sessions/SESSION-<ГГГГ-ММ-ДД>-<NN>.md` по шаблону:

```markdown
# Сессия <ГГГГ-ММ-ДД> #<NN>

## Над чем работали
- Шаг X — <названия задач>

## Сделано
- <конкретные пункты с путями файлов>

## В процессе / не закончено
- <что наполовину готово и где остановились>

## Решения
- <ключевые выборы за сессию>

## Блокеры / вопросы
- <что блокирует или что уточнить в следующий раз>

## Следующий шаг (для /start)
- <одно самое конкретное действие>
```

Заполнять честно и конкретно (пути, названия шагов): следующая сессия читает ТОЛЬКО этот файл + docs/10 + CLAUDE.md, а не переписку.

## 5. Подтвердить
Сказать пользователю: сессия сохранена, где лежит лог, и одной строкой «следующий шаг». Коротко. После `/stop` новую работу не начинать.
