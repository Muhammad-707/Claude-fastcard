# /start — начать / продолжить сессию

Ты работаешь над проектом **Fastcart** (e-commerce фронтенд). Делай по порядку, шаги загрузки контекста не пропускай.

## 1. Загрузить контекст
1. Прочитать `CLAUDE.md` (главные правила, Figma-first, стек).
2. Прочитать `docs/10_IMPLEMENTATION_PLAN.md` (план + чекбоксы + блокеры).
3. Посмотреть `.claude/sessions/` и прочитать **последний** `SESSION-*.md`, если есть. Если нет — это первая сессия, начать с Шага 0.

## 2. Сориентироваться
- Найти **первый шаг, который не `[x]`** в `docs/10` (сверху вниз). Если есть `[~]` (в работе) — взять его.
- Проверить раздел «Блокеры»: не начинать `[!]`, пока блокер не снят.
- Перечитать нужные docs для этого шага (например `docs/03` для API, `docs/08` для конкретной страницы, `docs/04` для токенов, `docs/11` для системных компонентов).
- **Figma-first:** если шаг строит UI — сначала убедиться, что Figma MCP доступен (docs/09). Недоступен → остановиться и сообщить пользователю, дизайн не выдумывать.

## 3. Короткий план (макс ~6 строк)
- Какой шаг плана берёшь.
- Какие файлы создашь/изменишь.
- Любое допущение/вопрос (спрашивать пользователя только если реально блокирует; иначе выбрать разумный дефолт и отметить).
Отметить шаг как `[~]` в `docs/10`.

## 4. Выполнить
- Сделать этот **один шаг** (или тесно связанную группу) до конца.
- Соблюдать правила CLAUDE.md: логика в хуках/thunks, API только через `shared/api` + Redux, тексты через `t()`, темы через CSS-переменные, без `any`, без отключения линтера, 3 состояния (loading/success/error).
- После кода прогнать `npm run type-check` и `npm run lint`; что сломал — починить.
- Нашёл новый пробел/блокер → дописать в «Блокеры» `docs/10` и продолжать с того, что можно.

## 5. Передача
- Когда шаг готов (или ставишь на паузу) — сказать, что сделано, и предложить запустить **`/stop`**, чтобы зафиксировать прогресс перед концом сессии.
- Сам лог сессии тут не пиши — это работа `/stop`. (Если пользователь резко уходит — можешь вызвать `/stop` сам.)
